# Governance Sandbox HTML multiline report note

When scenario authors add line breaks or short paragraphs to the proposal copy or decision memo, keep the HTML report readable instead of flattening everything into one block.

Recommended PM review cue:

- one scenario file with multiline proposal text,
- one generated HTML report that preserves paragraph/line-break structure,
- one validator pass before push.

That keeps markdown/html report work grounded in visible report quality instead of metadata-only progress.
