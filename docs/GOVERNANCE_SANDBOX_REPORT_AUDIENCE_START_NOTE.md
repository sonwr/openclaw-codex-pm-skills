# governance-sandbox report audience start note

Use this note when a governance-sandbox update needs the shortest replay path for scenario-file report audience metadata.

Keep the proof tight:

1. load one JSON or YAML scenario file,
2. set `report.audience` (or the documented audience aliases),
3. generate markdown/html report artifacts,
4. confirm the same audience string stays visible in the output bundle.

This keeps scenario-file input and report-file output coupled in one reviewable slice before wider web-demo work.
