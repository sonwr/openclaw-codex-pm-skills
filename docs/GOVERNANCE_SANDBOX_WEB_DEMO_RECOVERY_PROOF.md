# Governance Sandbox Web Demo Recovery Proof

When a governance-sandbox web-demo run breaks, recover in the smallest believable slice before widening scope again.

## Recovery order

1. Reopen the exact scenario fixture or preset input that failed.
2. Confirm the form still accepts the proposal, stakeholder presets, and report metadata without guessing hidden defaults.
3. Re-run the narrow path that should reveal one result card and one downloadable report bundle.
4. Capture which checkpoint passed last (input load, simulate action, result card render, report download).
5. Only after that proof exists, widen coverage to richer presets, polish, or UI copy.

## Status-line reminder

Prefer a short maintainer note that says what recovered, what still blocks, and which artifact proves the claim.
