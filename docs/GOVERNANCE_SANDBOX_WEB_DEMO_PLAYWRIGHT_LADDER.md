# Governance Sandbox Web Demo Playwright Ladder

Use this note when the first governance-sandbox web demo needs a short stability-first checkpoint ladder before adding broader browser coverage.

## Ladder

1. Prove the scenario input can be loaded with one replayable fixture.
2. Prove one result card renders with deterministic text.
3. Prove one report download path still works.
4. Only then widen to secondary form states, alternate fixtures, or multi-step browser flows.

## Why it exists

This keeps the first browser-demo slice aligned with stability, reproducibility, and step-by-step verification before scope expansion.
