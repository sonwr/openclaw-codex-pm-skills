# Governance sandbox scenario report artifact note

Use this note when a PM handoff should keep **one scenario file**, **one shared report bundle basename**, and the generated **JSON, Markdown, and HTML outputs** visible in the same sentence.

Keep the wording compact:

- name the imported scenario file first
- name the shared report bundle basename second
- list the JSON, Markdown, and HTML outputs third
- mention that the artifact paths are visible in the report metadata

This keeps scenario-file replay proof and report-output proof coupled in one reviewer-ready memo.
