# Governance Sandbox web demo download scope note

Keep the first governance-sandbox browser demo intentionally narrow:

1. one scenario input form,
2. one reviewable result card,
3. one obvious report download handoff.

Do not widen into multi-screen navigation, dashboard chrome, or secondary export variants until this form -> result card -> download path is stable, reproducible, and easy to replay.

This mirrors the current delivery order:

- scenario-file input first,
- markdown/html/json report output second,
- preset-aware wiring third,
- browser-demo polish after the report handoff is already believable.
