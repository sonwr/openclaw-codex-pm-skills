# Governance Sandbox scenario-to-report start note

Use this note when the smallest useful PM validation slice is **one scenario file -> one JSON/Markdown/HTML report bundle**.

Keep the scope tight:
- import one JSON or YAML scenario file;
- run one replay that emits JSON, Markdown, and HTML outputs;
- verify the artifact paths are visible in the report metadata before widening the workstream.

This keeps scenario-input proof and report-output proof coupled in a single reviewer-friendly loop.
