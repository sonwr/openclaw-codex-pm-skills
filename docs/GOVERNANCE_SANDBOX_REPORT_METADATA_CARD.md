# Governance sandbox report metadata card

Use this note when one imported scenario already generates a reviewable JSON/Markdown/HTML bundle and you want the metadata handoff to stay readable before widening the web demo.

Keep one reviewer-facing card that shows:
- generated-at timestamp
- scenario source path
- report bundle directory
- JSON/Markdown/HTML artifact paths

Validation stays small when the same metadata card confirms one imported scenario plus one generated JSON/Markdown/HTML report bundle before UI scope grows.
