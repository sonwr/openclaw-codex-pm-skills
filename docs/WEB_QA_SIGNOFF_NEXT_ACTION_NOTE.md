# WEB_QA_SIGNOFF_NEXT_ACTION_NOTE

When a web QA report already passes structural validation but signoff coverage is incomplete, add one explicit `next_action` line before calling the report handoff done.

Keep the note short:

1. what remains,
2. who owns it,
3. what artifact to reopen first.

This keeps strict-plus passes reviewable without inflating the rest of the signoff block.
