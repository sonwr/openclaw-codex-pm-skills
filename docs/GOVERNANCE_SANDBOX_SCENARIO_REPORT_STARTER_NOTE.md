# Governance Sandbox scenario report starter note

Keep the first governance-sandbox slice anchored to one imported scenario file and one visible JSON/Markdown/HTML report bundle.

- Start with a JSON or YAML scenario file that carries the proposal and stakeholder roster.
- Validate the CLI output against one shared report basename before expanding presets or web UI work.
- Treat web-demo follow-up as a later lane after the scenario-file + report-bundle proof is stable.
