# Governance sandbox scenario-sheet-bundle note

Use `scenario_sheet_bundle` when one wrapped JSON or YAML fixture should keep scenario input and report-ready output metadata together.

- Keep one scenario file first.
- Keep proposal + stakeholder input readable inside the wrapper.
- Keep report-title or bundle naming metadata in the same wrapped payload.
- Treat the alias as fixture ergonomics for the same reproducible scenario artifact path, not as a new report flow.
