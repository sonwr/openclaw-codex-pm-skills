# CLI summary report subject + owner + queue status note

Keep the saved summary handoff to one reviewer-facing line:

- name the report **subject** so the bundle has a visible topic
- name the **owner** so follow-up responsibility is explicit
- keep the current **reviewer queue** in the same line
- include the saved JSON / Markdown / HTML bundle status so the handoff is reopenable

Use this when the summary already passes validation and you want a short status sentence before commit/push.
