# WEB_QA_PLAYWRIGHT_NEXT_ACTION_PRIORITY_CHECK

Use this quick check before calling a strict-plus report replay-ready or owner-ready.

## Priority check

1. Start with the failed check ids in `Next action:`.
2. Confirm the sentence still points at the hotspot section or blocker that actually dominates the JSON metadata.
3. Prefer the first fix that restores stable target refs and evidence artifacts for the blocked lane.
4. If multiple blocker classes remain, name the owner or rerun lane before adding extra prose.
5. Keep the sentence short enough to paste into a handoff without reopening the full report.

## Pass signal

The next-action sentence should answer three questions in one read: which checks failed, which lane goes first, and what proof should be restored on the rerun.
