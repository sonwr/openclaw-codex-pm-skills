# Governance sandbox scenario report metadata start note

Use this note when a PM handoff needs the smallest reproducible path for reopening governance-sandbox scenario artifacts with stable metadata.

## Fast path

1. Open `docs/GOVERNANCE_SANDBOX_SCENARIO_REPORT_QUICKSTART.md` for the fixture-first command.
2. Reopen `docs/GOVERNANCE_SANDBOX_REPORT_METADATA_CARD.md` to confirm which report fields should stay visible in JSON, markdown, and HTML artifacts.
3. Keep the first review on one scenario file and one shared basename before widening into demo/UI follow-up.

## Why it matters

- Stable metadata makes report bundles easier to hand off between PM, docs, and demo lanes.
- Shared artifact naming reduces confusion when scenario basenames differ from the default alias files.
- Metadata-first review keeps scenario-file provenance visible before anyone starts debating UX polish.
