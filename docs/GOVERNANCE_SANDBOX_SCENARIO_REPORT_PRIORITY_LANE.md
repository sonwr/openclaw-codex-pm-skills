# Governance Sandbox scenario-report priority lane

Keep the PM lane narrow until one imported scenario file reliably produces the same JSON, Markdown, and HTML report bundle.

Priority order:

1. Scenario file import proof (`.json` / `.yaml`)
2. Markdown and HTML report generation proof
3. Preset-backed stakeholder coverage
4. Result-card/web demo widening
5. Demo GIF and polish

Use this note when a workstream is tempted to widen UI scope before the scenario-input and report-output contract is stable.
