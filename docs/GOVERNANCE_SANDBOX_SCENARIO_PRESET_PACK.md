# governance-sandbox scenario preset pack

Use this note when a governance-sandbox scenario file needs to stay readable while proving preset-backed stakeholder coverage before the first wider demo.

## What to keep stable
- Start with one scenario file that contains the proposal plus stakeholder roster.
- Prefer trait presets (`dao`, `delegates`, `contributors`, `investors`, `community`) before custom free-form traits.
- Keep the first proof scoped to one named JSON + Markdown + HTML report bundle.

## Why it matters
This keeps scenario-file input work tied to reviewable report artifacts, so preset expansion does not drift away from reproducible evidence.
