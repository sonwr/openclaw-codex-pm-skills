# Governance Sandbox web demo result-card audit

Use this quick audit before widening the first governance-sandbox web demo beyond one form and one result card.

1. One visible action: the form should lead to exactly one primary result card.
2. One proofable output bundle: the card should point to JSON/Markdown/HTML report artifacts or their download path.
3. One stable replay target set: each primary interaction should keep deterministic refs/selectors for browser reruns.
4. One recovery path: blocked states should say whether the next move is rerun, repair, or hold.
5. One honest scope line: do not describe the slice as a full governance product when it is still a demo path.
