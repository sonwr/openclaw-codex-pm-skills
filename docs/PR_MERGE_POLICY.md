# PR Merge Policy (Web QA)

For web-service related changes, a PR is merge-ready only when all conditions below are met.

## Required conditions

1. Regression count is **0**.
2. QA checklist is attached and completed:
   - functional checks: 5
   - visual checks: 3
   - off-happy-path checks: 2
3. Evidence is attached:
   - screenshots for relevant visual checks
   - concise validation log

## Block conditions

- Any regression > 0
- Missing screenshots for visual claims
- Missing QA checklist sections
