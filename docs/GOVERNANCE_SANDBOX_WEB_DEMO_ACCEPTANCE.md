# Governance Sandbox Web Demo Acceptance

Use this note when the first governance-sandbox web demo needs a compact acceptance bar before broadening the UI scope.

## Acceptance bar

1. One scenario fixture can be loaded without manual cleanup.
2. One result card renders stable proposal, audience, and recommendation text.
3. One report artifact path remains visible from the same card or follow-up action.
4. One replay note names the exact fixture and validation command used for the proof.

## Why it exists

This keeps the first web-demo proof aligned with UI/UX clarity, deterministic replay, and staged Playwright verification instead of widening the surface too early.
