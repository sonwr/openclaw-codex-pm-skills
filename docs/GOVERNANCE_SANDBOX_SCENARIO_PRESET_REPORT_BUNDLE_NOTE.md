# Governance sandbox scenario + preset report bundle note

Keep governance-sandbox scope intentionally small and proof-oriented:

1. start from one scenario file as the source of truth
2. keep one preset-backed stakeholder mix visible in that same fixture
3. regenerate one shared markdown/html/json report bundle from the same run
4. mention the scenario path, preset evidence, and artifact bundle together in the handoff

This keeps the current delivery ladder honest: scenario file input first, report bundle proof second, preset coverage third, and wider web-demo work later.
