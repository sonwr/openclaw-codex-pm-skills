## Scope check

Keep the change scoped only if all of these are true:

1. The demo still starts from a replayable scenario input, not ad-hoc inline state.
2. The result view still exposes the recommendation plus stakeholder response evidence.
3. The proof path still names at least one stable browser check and one report artifact.
