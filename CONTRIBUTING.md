# Contributing to openclaw-codex-pm-skills

Thank you for contributing.

## Scope

This repository accepts contributions that improve:

- PM skill quality
- portability across OpenClaw/Codex-style workflows
- documentation clarity
- example realism

## How to contribute

1. Fork the repository
2. Create a focused branch
3. Make one cohesive change
4. Add/update examples when behavior changes
5. Open a PR with clear rationale

## Skill authoring expectations

Each skill should include:

- purpose
- when-to-use guidance
- workflow steps
- deterministic output format
- practical constraints and caveats

## PR template expectations

In your PR description, include:

- What changed
- Why it changed
- Before vs after behavior
- Example output snippet
- Compatibility concerns (if any)

## Review standards

PRs may be rejected if they are:

- too broad,
- unclear in output expectations,
- tightly coupled to one vendor runtime,
- or missing practical examples.

## Code of collaboration

- Be precise
- Be constructive
- Keep discussions evidence-based
