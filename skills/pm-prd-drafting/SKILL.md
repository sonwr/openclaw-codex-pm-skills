# PM PRD Drafting

## Purpose
Draft a practical PRD with explicit scope, constraints, risks, and acceptance criteria.

## Use when
- Converting validated ideas into execution-ready specs
- Aligning product, design, and engineering

## Workflow
1. Define problem and target user
2. Clarify goals and non-goals
3. Define scope (in/out)
4. Write functional requirements
5. Add success metrics
6. Add risks and mitigations
7. Define acceptance criteria

## Output
- PRD summary (1 page)
- Requirement checklist
- Acceptance criteria table
